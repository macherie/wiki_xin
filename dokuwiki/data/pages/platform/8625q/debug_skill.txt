====== 8625Q 平台使用调试技巧 ======

**这里记录了8625Q平台一些实用的调试技巧**


===== Get dump  =====

Get dump :

NV set ting:

905 0 (enter dload after a crash)

4399 1 (detect hardware reset)


The correct way to test the behavior after a crash would be:
  - Set NV 905 to 0.
  - Reset phone.
  - [opt] Phone is running, send the command "send_data 75 37 03 00" in QXDM. (This triggers an ERR_FATAL on the phone)
  - [opt] Phone should reboot and enter dload mode on its own.
  - Run the QPST Memory Debug tool. The phone should show up as being in "download mode"
  - Follow normal USB log collection procedure (basically click the "get regions" button, then "save to:")


===== Get Build ID  =====

<code bash>
adb shell cat /sys/devices/system/soc/soc0/build_id
</code>

===== 查看有谁占这 wakelocks =====

用串口抓 cat /proc/wakelocks,然后用下面命令过滤

<code bash>
cat wakelocks.cap | awk '{print $1 " " $5}' | grep -v " 0" | grep -v active
</code>

（过滤出第5位不是0， 不是列名的行）
